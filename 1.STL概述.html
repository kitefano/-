STL概述

 

STL是Standard Template Library 中文译为“标准模板库”

 

STL特点:

STL的主要功能就是用于数据处理

是数据结构和算法的分离。这种分离确实使STL变得非常通用,程序员可以在不用思考算法就可以使用数据

c语言中的malloc() 没有泛型编程

  • 通过 void*类型的指针强转
  • sizeof 获取长度大小

在STL中体现了泛型化程序设计

  • 思想（generic programming）
  • 需求（requirements）
  • 概念（concept）
  • 模型（model）
  • 容器（container）
  • 算法（algorithmn）
  • 迭代（iterator）

STL中的组件

  • 容器（Container），是一种数据结构，如list，vector，和deque ，以模板类的方法提供。为了访问容器中的数据，可以使用由容器类输出的迭代器；
  • 迭代器（Iterator），提供了访问容器中对象的方法。例如，可以使用一对迭代器指定容器中的一定范围的对象。迭代器就如同一个指针。迭代器也可以是那些定义了operator*()以及其他类似于指针的操作符地方法的类对象；
  • 算法（Algorithm），是用来操作容器中的数据的模板函数。例如，STL用sort()来对一个数组容器中的数据进行排序，用find()来搜索一个list中的对象，函数本身与他们操作的数据的结构和类型无关，因此他们可以在从简单数组到高度复杂容器的任何数据结构上使用;
  • 仿函数 Function object，函数对象（function object)
  • 迭代适配器（Adaptor）
  • 空间配制器（allocator）也是我们的常用的内存对象 1.对象的创建与销毁 2.内存的获取与释放

序列式容器

序列式容器（Sequence containers），每个元素都有固定位置，array,vector、deque、list；

  1. array 数组

  2. vector: 动态数组

  3. deque: 双端队列 (double-ended queue)的缩写,连续存储的指向不同元素的指针所组成的数组

  4. list：双向链表 (double-linkedlist) 由节点组成的双向链表，每个结点包含着一个元素

     

关联式容器

关联式容器（Associated containers）,元素位置取决于特定的排序准则，和插入顺序无关

  • set 会自动进行排序， 不会出现重复的元素
  • multiset 多重集合(multiset) 允许存在两个次序相等的元素的集合
  • map 映射(map) 由{键，值}对组成的集合，以某种作用于键对上的键的值排列
  • multimap 多重映射(multimap) 允许键对有相等的次序的映射

底层是由内部由红黑树(RB-tree）实现,可以高效的进行查找

仿函数

仿函数(Function) //相当于封装好的函数指针,可以将一个函数通过一个变量存储起来使用

适配器容器

优先队列(priority_queue) // 底层实现用的是一个堆,可以讲一组数据按特定的方式进行排列,每次能从中取出最大值

智能指针

unique_ptr 拥有独有对象所有权语义的智能指针,一个指针指向一段内存

shared_ptr 拥有共享对象所有权语义的智能指针 ,可以多个指针指向同一块内存空间

auto_ptr 这个智能指针已经准备被弃用了c++17

智能指针的共同特性,当这个内存没有任何指针指向的时候,会自动释放这块内存空间
